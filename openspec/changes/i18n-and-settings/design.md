## Context

AgentBoard Desktop 是一个 Electron macOS 应用，当前所有界面文案直接硬编码为中文字符串，散落在 `widget.html`、`magiclink.html`、`tray.ts`、`index.ts` 等多个文件中。项目即将开源，需要支持英文界面。同时缺少统一设置入口，语言偏好需要一个窗口来配置。

## Goals / Non-Goals

**Goals:**
- 所有用户可见文案通过 i18n 模块读取，支持 `zh`（中文，默认）和 `en`（英文）
- 新增设置窗口，提供语言切换，切换后所有已打开界面实时更新
- 语言偏好持久化，重启后保持上次设置
- About 对话框项目主页更新为个人 GitHub 仓库地址

**Non-Goals:**
- 不支持第三方语言包或动态添加语言
- 不做 RTL 支持
- 不做日期/数字格式本地化（数字格式保持现有逻辑）

## Decisions

### D1：i18n 模块放在主进程，渲染进程通过 preload 获取翻译

**选择**：在 `src/main/i18n.ts` 集中管理所有字符串资源和当前语言状态，渲染进程通过 preload 暴露的 `t(key)` 函数获取翻译文本。

**备选**：在每个渲染进程各自管理翻译（各自 import JSON）。

**原因**：主进程持有 `electron-store` 实例管理持久化，语言切换时通过 `webContents.send('i18n:changed', locale)` 广播，渲染进程统一响应重渲染，避免各页面各自持有状态导致不一致。

### D2：语言切换触发全量重渲染

**选择**：语言切换后，主进程向所有渲染窗口广播 `i18n:changed`，渲染进程调用统一的 `applyLocale()` 函数，通过 `data-i18n` 属性批量替换 DOM 文本。

**备选**：切换后关闭并重新创建所有窗口。

**原因**：无需重建窗口，用户体验更流畅；DOM 批量替换实现简单，文案节点有限，性能无影响。

### D3：设置窗口为独立 BrowserWindow

**选择**：`settings.html` 作为独立的轻量窗口（360×240），由系统菜单 Settings 菜单项触发，设为 `alwaysOnTop: false`，`resizable: false`。

**原因**：与登录窗口模式一致，隔离渲染上下文，实现简单。

### D4：翻译键使用平铺命名空间

**选择**：`{ "tab.people": "用户榜", "tab.teams": "团队榜", ... }` 平铺结构。

**原因**：文案数量少（< 50 条），无需嵌套；平铺对 TypeScript 类型推导友好。

## Risks / Trade-offs

- **硬编码遗漏**：散落的中文文案较多，可能存在遗漏。→ 通过 grep 全量扫描 `[\u4e00-\u9fff]` 覆盖
- **设置窗口多开**：用户多次点击菜单项可能重复创建窗口。→ 用单例守卫（窗口未销毁则 focus）
- **重渲染闪烁**：批量替换 DOM 文本可能出现短暂闪烁。→ 文案节点少，实测无感知，无需特殊处理
